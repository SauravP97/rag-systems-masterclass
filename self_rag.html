1. The Problem with Vanilla RAG

Standard Retrieval-Augmented Generation (RAG) improved LLMs by giving them external knowledge. However, it suffers from critical flaws:

  • Unnecessary Retrieval: Fetches documents for every query, wasting time on simple questions.
  • Irrelevant Context: If retrieval fails, the generator hallucinates based on bad context.
  • Lack of Attribution: No guarantee the generated response is actually supported by the retrieved text.

2. The Self-RAG Solution

Self-RAG trains a single LLM to self-reflect on its generation process on-demand using special reflection tokens.

[Retrieve]

Decides if external info is needed.
Values: Yes, No, Continue

[IsREL] (Relevance)

Evaluates if the document contains useful info.
Values: Relevant, Irrelevant

[IsSUP] (Supported)

Checks for hallucinations based on context.
Values: Fully, Partially, No support

[IsUSE] (Utility)

Holistic score of how useful the response is.
Values: 1 to 5
