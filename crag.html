Corrective RAG

A Master Class on building robust LLM applications by introducing self-reflection, evaluation, and active knowledge refinement into the retrieval pipeline.

The Hallucination Bottleneck

Standard RAG blindly trusts the retrieved documents. If the retriever fetches irrelevant or incorrect data, the LLM is forced to rely on flawed context, leading to confident hallucinations.

The CRAG Architecture

CRAG introduces a Retrieval Evaluator. It scores the retrieved documents and routes the data flow based on confidence levels. Experience the logic using the interactive simulator below.

Select a Retrieval Scenario

Click one of the confidence levels above to see how the CRAG evaluator routes information to prevent hallucinations.

Action A: Correct

Triggered when the evaluator is highly confident. The documents are relevant, but may contain noise. CRAG sends them to the Knowledge Refinement module to extract only the most crucial facts before generation.

Action B: Incorrect

Triggered when retrieval fails entirely. To prevent hallucinations, CRAG discards the local documents. It automatically generates keywords and triggers a Web Search to find accurate, external knowledge.

Action C: Ambiguous

Triggered when confidence is mixed. CRAG combines strategies: it refines the helpful parts of the local retrieval while simultaneously running a Web Search to fill in the missing context.
